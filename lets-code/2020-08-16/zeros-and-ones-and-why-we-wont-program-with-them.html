Zeros and ones, and why we won't program with them

Most media depictions of programming use images with 0s and 1s. And understandably so, because at its core that is the extent of a computer's "alphabet": 0s and 1s. Because it has only two characters, it is called a binary system. Believe it or not, with just those two characters you can represent any message. Let's see an example.

Imagine that you and a friend wanted to communicate over a long distance. Each of you have two laser pointers; one with a blue beam and the other with a red beam. When a programmer changes the way something, like a letter, is represented we call that encoding. Then they made programs that translate from the higher level languages to the 0s and 1s (aka machine code) that the computer understands. No more programming in 0s and 1s for us mere mortals.

Today you were introduced to several new concepts. The most important is that there are programs which translate from higher level languages to machine code. These programs come in several different forms. In our case we will almost exclusively use a runtime engine which, as the name implies, will run or execute our code. The runtime engine in turn has an interpreter within it that understands the language that we will use.
